  Actions and GitLab CI templates.
- Python source imports remain an unsupported contributor/testing mechanism.

  mypy, unit/integration tests, and compilation passed on the RC evidence
  branch before publication.
- GitHub and GitLab exact-value secret audits were completed by the operator on
  2026-07-21 without recording secret values in the repository.

## RC known gaps

These gaps are accepted for `1.0.0-rc.1` and must be completed or explicitly
waived before final `v1.0.0`:

- GitHub oversized raw-diff HTTP 406 has not been exercised live. Unit coverage
  verifies the fail-closed message and no-bundle behavior.
- GitHub manifest-finalization head movement has not been hit live. Two
  recorded live attempts missed the short finalization window and completed
  successfully; unit coverage verifies the three-read stale-head path.
- Some GitHub and GitLab lifecycle variants remain partial, including positive
  changed-body in-place update and unrelated line movement behavior.
- GitLab hostile-MR override/forgery variants remain partial, though protected
  credential withholding, forwarding isolation, and symlink containment passed
  against P0.

## Upgrade

Pin CI templates to this tag and keep image references digest-pinned. Do not use
mutable image tags as the release identity.

GitHub users should require the `gate` check on protected branches if blocking
review findings must prevent merge. GitLab users should keep "Pipelines must
succeed" enabled for merge blocking.
