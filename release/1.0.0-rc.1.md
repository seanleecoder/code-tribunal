# Code Tribunal 1.0.0-rc.1

This release candidate published an earlier container/template set for
operator validation. It is **superseded** for final `v1.0.0`: runtime work after
`15d424f` (mock scenarios, gate `run_id` binding) requires a rebuilt image pair,
and `release/release-inputs.json` is now `draft` until the live-evidence matrix
passes against that pair. See [`1.0.0.md`](1.0.0.md) and
[`docs/history/evidence/README.md`](../docs/history/evidence/README.md).

## RC known gaps

Live evidence is now classified by tier in the
[evidence index](../docs/history/evidence/README.md). Only release-gating gaps
must be completed or explicitly waived before final `v1.0.0`; regression-covered
rows are proven by `make quality` and do not block the release.

Release-gating gaps still open:

- Positive changed-body in-place update has not been demonstrated live on either
  platform (probes fell back to summary-only or resolved the old thread). It is
  unit-covered and can now be reproduced token-free with the deterministic mock
  reviewer (see the RC runbook).
- GitLab hostile-MR trusted image/config override and forged-gate rejection at a
  credential-bearing boundary remain partial, though protected credential
  withholding and forwarding isolation passed against P0.

Reclassified as regression-covered (live confirmation optional, not a release
gate):

- GitHub oversized raw-diff HTTP 406 — fail-closed message and no-bundle behavior
  verified by unit tests; the race windows were never reproducible live.
- GitHub revision-race boundaries including manifest-finalization — the three-read
  stale-head path is unit-verified.
- SPEC-31 symlink containment (all variants) and SPEC-33 gate/config artifact
  integrity — verified by the unit suite.

## Upgrade

Pin CI templates to this tag and keep image references digest-pinned. Do not use
mutable image tags as the release identity.

GitHub users should require the `gate` check on protected branches if blocking
review findings must prevent merge. GitLab users should keep "Pipelines must
succeed" enabled for merge blocking.
