# Code Tribunal 1.0.0 (draft release notes)

These notes become final when `v1.0.0` is tagged. Until then,
`release/release-inputs.json` remains `status: draft` and must not be treated as
a published 1.0 artifact set.

## Release identity (fill at tag time)

- Runtime source `R`: _pending freeze_
- Release commit `P`: _pending pin commit_
- Tag: `v1.0.0`
- Base image: _digest from publication of R_
- Reviewer image: _digest from publication of R_
- External manifest: build with
  `python scripts/build_release_manifest.py --tag v1.0.0 --runtime-source "$R" --release-commit "$P" --out /tmp/release-manifest.json`
  then validate with `python scripts/check_release_manifest.py /tmp/release-manifest.json`

## Scope

- Supported distribution: digest-pinned OCI images plus GitHub Actions and
  GitLab CI templates.
- Python source imports remain an unsupported contributor/testing mechanism.
- Cursor is experimental and outside the 1.0 live-evidence matrix.
- Semantic grouping remains YAML-only, disabled, and outside the 1.0
  compatibility guarantee; env overrides are rejected.

## Operator completion checklist

1. Freeze `R` after this readiness-hardening branch lands on `main`.
2. Publish attested base+reviewer images from exactly `R`.
3. Re-run the release-gating live matrix in
   [`docs/history/evidence/README.md`](../docs/history/evidence/README.md).
4. Mark cited evidence records `Status: passed` with matching
   `Release-runtime-source` / digests (or explicit waiver lines).
5. Repin templates + set `release-inputs.status` to `active`; refresh hashes.
6. Cut `P`, move `CHANGELOG` `[Unreleased]` to `[1.0.0] - <date>`, tag
   `v1.0.0`, attach the external manifest and checksum to the GitHub release.

## Upgrade / rollback

Follow [`docs/operations.md`](../docs/operations.md). Rollback means restoring
the previous immutable template/image set (for example `v0.4.0`), not pulling a
mutable tag.

## Known limitations at draft time

- Live evidence against the final image pair is incomplete (see evidence index).
- Network egress is not enforced at the container/runner boundary.
- Symlink-containing repositories are rejected during snapshot preparation.
- GitHub external forks are skipped; GitLab hostile-MR deployment evidence must
  be completed before broad credential-isolation claims.
